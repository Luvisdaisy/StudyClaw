# Streamlit 知识库重构计划

## 背景

将现有客服机器人项目扩展为类似 **NotebookLM** 的个人知识库。

**核心功能范围**：

- 文档上传与管理（PDF, TXT, Markdown等）
- 基于文档的 AI 问答（RAG）

---

## 技术方案

### 架构设计

```
┌─────────────────────────────────────────────────┐
│                  Streamlit UI                   │
│  ┌──────────────┐  ┌─────────────────────────┐│
│  │  侧边栏       │  │  主区域                  ││
│  │  - 文档上传   │  │  - Tab: 知识库问答       ││
│  │  - 文档列表   │  │  - Tab: 文档管理         ││
│  │  - 删除文档   │  │                         ││
│  └──────────────┘  └─────────────────────────┘│
└─────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────┐
│              RAG Service Layer                  │
│  - document_processor (新增)                    │
│  - vector_store (扩展支持多 collection)         │
│  - rag_service (复用)                           │
└─────────────────────────────────────────────────┘
```

---

## 实施步骤

### 步骤 1：重构 app.py

- 添加 `st.sidebar` 侧边栏用于文档管理
- 使用 `st.tabs` 切换问答/管理视图
- 实现文档上传组件（st.file_uploader）

### 步骤 2：新增文档处理模块

- 创建 `rag/document_processor.py`
- 支持 PDF（pdfplumber）、TXT、Markdown 解析
- 自动分块、生成 embeddings

### 步骤 3：扩展向量存储

- 修改 `rag/vector_store.py`
- 支持按用户/文档隔离的 collection
- 实现文档级别的增删改查

### 步骤 4：集成 RAG 问答

- 复用现有 `rag_service.py`
- 支持选择特定文档进行问答
- 显示来源引用

---

## 关键文件修改清单

| 文件                        | 操作     | 说明                            |
| --------------------------- | -------- | ------------------------------- |
| `app.py`                    | 重构     | 侧边栏 + Tab 布局 + 文档上传 UI |
| `rag/document_processor.py` | **新增** | 文档解析（PDF/TXT/MD）          |
| `rag/vector_store.py`       | 扩展     | 多文档 collection 支持          |
| `rag/rag_service.py`        | 调整     | 支持指定文档范围的检索          |
| `utils/file_handler.py`     | 扩展     | 复用为统一文件处理              |

---

## 验证方式

1. 运行 `uv run streamlit run app.py` 启动应用
2. 侧边栏上传 PDF/TXT 文档
3. 切换到问答 Tab，测试基于上传文档的问答
4. 检查文档列表和删除功能
